# 大方向上新思路
1. 打基础
    - HTTP/TCP/IP/TLS协议
    - Linux操作系统、日志分析、权限机制
    - Python编程、脚本自动化
    - Web框架（如 Spring、Django）理解请求处理流程

2. 研究漏洞原理
    - 挑选经典漏洞
    - 读源码，看漏洞产生原因
    - 分析漏洞触发条件、数据流、调用链

3. 复现漏洞
    - 搭建漏洞环境（Docker/VM）
    - 捕获异常流量（Wireshark / Zeek）
    - 写 PoC / exploit 脚本
    - 总结分析报告

4. 做真实研究
    - 审计开源项目（寻找潜在漏洞）
    - 开发自动化工具（漏洞扫描、流量分析）
    - 形成个人研究成果，可放简历或 GitHub

# 近期思路
一、搭建个人安全研究实验室
```
SecurityLab/
├─ web-labs/         # DVWA, Juice Shop, Struts, Log4j等
├─ tls-labs/         # OpenSSL漏洞、Heartbleed、TLS异常流量
├─ cve-research/     # CVE复现环境
├─ tools/            # PoC脚本、漏洞利用工具
└─ notes/            # 漏洞分析报告、实验记录
```

工具&技术栈
  - Docker / VirtualBox / VM
  - Wireshark / tcpdump / Zeek
  - Python / Bash / Go（写 PoC / 自动化工具）
  - IDE（VSCode / IntelliJ / PyCharm，方便打断点调试）

二、实验室日常流程
1. 搭建环境
    - 安装目标软件或漏洞版本
    - 确保可以触发漏洞

2. 抓包分析
    - Wireshark / Zeek 捕获流量
    - 观察异常数据、请求、响应

3. 阅读源码
    - 打断点、跟踪数据流
    - 看变量变化、函数调用链

4. 写 PoC 脚本
    - Python / Bash / Go 编写自动化触发漏洞
    - 捕获异常或执行 exploit

5. 写漏洞分析
    - 漏洞类型、触发条件
    - 利用方法 / payload
    - 修复或防御思路
    - 可视化流程图或抓包截图

6. 总结与抽象
    - 形成知识体系
    - 记录在 notes/ 目录，方便以后复盘或面试展示
