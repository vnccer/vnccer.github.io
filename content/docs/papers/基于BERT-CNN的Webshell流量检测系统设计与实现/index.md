# 一、精读随笔
## 1.1 特征提取
>预处理：使用 wireshark 对 pcap 包进行解析，按照**请求方法**、**URL**、**Host**、**User-agent**等字段保存为 CSV 格式。

对预处理模块中得到的 CSV 格式数据进行选择，选择的字段有：**请求 payload**、**Cookie**、**Accept-Language**、**User-Agent**。

## 1.2 模型架构
>BERT 编码 + CNN 检测

- BERT 编码：
  - 输入层：将离散的文本字符转化为密集、带有结构信息的初始向量，每条输入的流量特征文本会被融合为三种不同的嵌入向量：
    - Token Embedding(词嵌入)：数据前[CLS]作为开始，[SEP]作为结束
    - Segment Embedding(段嵌入)：数值设为0
    - Position Embedding(位置嵌入)：向量根据序列长度依次递减，表示特征位置信息
  - Transformer-encoder层：实现“联系上下文”深度双向特征提取
    - 多头注意力机制：每个单词向量都计算其与其他所有单词的权重
    - 前馈神经网络：与多头注意力机制采用**残差结构连接**，保证输入的信息完整地传输到下一层中
  - 输出层：
    - 取输出层所有位置的词向量接入创新的 CNN 模型
    - 对长度不足 256 个词的流量特征实施补零，超长的阶段，最终输出一个维度固定为 (256, 128)的高维特征矩阵。

- 双层 CNN 检测模型：
  - 输入层：接收来自 BERT 编码模块生成的特征矩阵，每条流  - 量数据的标准维度为(256, 128)
  - 一维卷积层：卷积核大小为3，维度为64
  - 最大池化层：池化窗口大小为3
  - 一维卷积层：卷积核大小为3，维度为128
  - 平均池化层：从宏观角度对所有特征计算平均值，使模型平滑，同时将二维矩阵压缩拉长为一维向量，以便后续分类
  - 暂退层（Dropout Layer）：随即舍弃率为0.5
  - 全连接输出层：激活函数为 Sigmoid，将向量彻底降低到 1 维，生成一个区间在[0, 1]之间的概率值，判定流量正常还是 Webshell。



## 1.3 局限性
1. BERT 模型需要减少生成向量的时间
2. 目前不能从加密流量中检测 Webshell
# 二、难点记录
## 2.3.2 编码过程
**问题一：**如何理解“首先根据单词在**词库**中的**索引**生成**词嵌入向量**”？
- 词库：BERT 模型在训练前，科学家会塞一本巨大的字典，有常见的单词、符号。
- 索引：比如`[CLS]`索引可能是`101`，`=`索引可能是`29`。
- 词嵌入向量：
    计算机无法理解`29`代表`=`的含义。需要把单一的*数字*变成*数字组合*（包含丰富语义特征）。
    本文采用 Tiny-BERT 模型，特征向量维度是 128 维。那么，每个单词都会变成一个包含 128 个小数的数组（例如`[-0.997, 1.135, -3.045, ...]`）。
- 那么这里的过程就是，当拿到单词索引后，根据索引，把 128 个数字搬出来。


**问题二：**如何理解**段嵌入向量**所有值为0
- 首先，BERT 中，任何一个单词的最终特征都由三个向量相加得到的：
$$\text{最终输入向量} = \text{词嵌入 (Token)} + \text{段嵌入 (Segment)} + \text{位置嵌入 (Position)}$$
- **段嵌入向量**：BERT 最初设计，任务要输入两段话，如：回答问题需要**问题+答案**，下一句预测任务需要**前一句+后一句**，为了让模型分清两段话，BERT 引入“段嵌入”。
  - 第一段话的单词，段嵌入全给`0`
  - 第一段话的单词，段嵌入全给`1`
- 而本文的 Webshell 流量检测中，任务是“编码”、“分类”。模型每次只读入一条流量数据，既然没有第二句话，那么这条数据里的所有单词全归类为“第一段”，段编码就全为`0`。这意味着不需要区分段落，相当于全填`0`的无效向量。


**问题三：**如何理解**位置嵌入向量**根据序列长度依次递增？
- **位置嵌入向量**：BERT 的核心架构是 Transformer。传统的循环神经网络（RNN/LSTM）是一个字一个字顺序读入，天然知道字的先后顺序，但 Transformer 为了追求超高速的并行计算，把一句话里的所有字同时丢进模型处理，如：“我吃饭”和“饭吃我”是一模一样的。
- 为了解决缺陷，模型给每个字打上“位置标签”：
  - 第一个位置（`[CLS]`标志位）：编号为`0`
  - 第二个位置（如`cmd`）：编号为`1`
  - 第三个位置（如`=`）：编号为`2`


**问题四：**如何理解**多头注意力机制**输出对应的**特征词向量**？
- 进入第 3 步前，第 2 步拼好的 3 个嵌入向量（词、段、位置）已经相加融合成一个初始向量，此时模型知道这个词是谁、在什么位置。但向量目前是孤立、死板的。如：流量里的`id`，模型只知道自己叫`id`，但不清楚是正常网页脚本调用，还是被黑客来执行 Linux 的`id`命令。

- **注意力**：每个单词初始向量都乘以三个不同的随机矩阵，衍生出：查询向量$Q$、键向量$K$、值向量$V$，模型通过$Q$和$K$的内积计算，能算出任意两个词之间的相关性（权重），计算公式就在论文中：
$$Attention(Q,K,V)=Softmax\left(\frac{QK^{T}}{\sqrt{d_{k}}}\right)V$$

-**多头**：意味着模型不止算一次注意力，而是同时拉出好几组不同的$Q、K、V$矩阵（比如8个头）一起算，就像针对一条黑客流量，请 8 个不同领域专家会诊，1 号头：盯着语法结构，2 号头：盯着高危敏感词，3 号头：盯着位置距离...

- 经过**多头注意力机制**，输出的**特征词向量**不再是死板的初始向量，此时输出的`id`的向量里，融合了上下文`cmd`和`=`的信息，变成了：一个跟在`cmd=`后面，大概率是黑客用来探路执行系统命令的`id`词向量。


**问题五：**如何理解[CLS]标志位所对应的词向量，能够直接输入一个**多层感知机**进行分类？
- 意思是：可以把`[CLS]`这个自带全句总结的特征向量，直接丢进一个最普通的标准神经网络里去判断它是正常流量还是恶意 Webshell。
- 多层感知机（Multi-Layer Perceptron, MLP）是最传统、最基础的“全连接深度神经网络”：它能输入一串数字（向量），中间经过全连接层（Dense层），最后输出一个结果。
- 因为 BERT 内部的**多头注意力机制**，一句话里的每一个词都跟`[CLS]`发生关联并把信息分享给它，当 BERT 吐出最终结果时，`[CLS]`对应那个 128 维向量，里面高度浓缩了整条流量报文的核心意图。MLP 只要读懂这个“总结报告”，就能判断这条流量是不是 Webshell。

第二处： **2.3.2 编码过程**
在遇到词库中不存在的单词时会使用#号作为通配符进行匹配分词，如‘ipconfig’会被分割为‘##config’这里写的有问题。
如果分割，应该是['ip', '##config']
