# 一、整体流程概述
## 1.1 准备仓库
github中创建名为`用户名.github.io`的公开仓库，且仓库为空（不要初始化README）

## 1.2 配置Hugo项目
在本地Hugo项目根目录中，打开`hugo.toml`，将baseURL指向`https://用户名.github.io/`
确认`theme`指向`hugo-book`

## 1.3 自动化部署（Github Actions）
根目录创建`.github/workflows/`，且此目录中新建文件`deploy.yml`，填入Hugo官方推荐的Action配置（Checkout -> Setup Hugo -> Build -> Deploy to GitHub Pages）

## 1.4 开启Github pages
将本地代码推送到github仓库
进入github仓库的settings -> pages
build and deployment -> source中选择Githubactions

## 1.5 推送
当向仓库push代码时，github actions自动编译并更新网页
过个2分钟，就可以访问

# 三、github部署
## 3.1 初步部署
```POWERSHELL
cd D:\4work\2note\website666\rocky666\
# 关联远程仓库
git remote add origin https://github.com/vnccer/vnccer.github.io.git

# 添加文件到暂存区
git add .

# 设置邮箱
git config --global user.email "keepp955@gmail.com"

# 设置用户名
git config --global user.name "vnccer"

# 进行第一次本地提交
git commit -m "initial commit"

# 检查当前分支名（mian or master）
git branch

# 网络走梯子，修改代理端口是 7897
git config --global http.proxy http://127.0.0.1:7897
git config --global https.proxy http://127.0.0.1:7897

# 由于github中勾拳了生成README.MD或LICENSE文件，git不允许直接覆盖掉远程已有的内容
# 因此需要强制合并远程的初始化文件，弹出vim模式的合并说明，esc，:wq
git pull origin main --allow-unrelated-histories

# 完成网页发布
git push -u origin main
```

## 3.2 半自动化部署
vscode中创建`rocky666/.github/workflows/`，且在`workflows/`中新建文件`hugo.yml`，内容复制官方文档https://github.com/vnccer/vnccer.github.io/new/main?filename=.github%2Fworkflows%2Fhugo.yml&workflow_template=pages%2Fhugo

接着在github仓库setting中，修改Actions，General，Workflow permissions 修改为读写权限
在Pages中，将建造和部署下，将源修改为Action

接着在终端执行任务，将脚本发送给Github
```POWERSHELL
git init
git remote add origin https://github.com/用户名/用户名.github.io.git
git add .
git commit -m "添加自动化部署脚本"
git push -u origin main
```
  - `git remote add origin xxxURL`是用git建立自己网站的远程连接，远程仓库的代号为`origin`
  - `git push -u origin main`告诉git，以后本地的main分支，默认对应远程`origin`仓库里的`main`分支，这个命令成功运行过一次，git就会记住
  - 查看远程仓库名：`git remote`
  - 查看关联的github详细地址`git remote -v`，其中`fetch`指从哪里拉取代码，`push`指推送到哪里
  - 查看分支追踪关系`git remote show origin`，

以后无论本地更新什么都是**三板斧**
```powershell
cd D:\4work\2note\website666\rocky666\
git add .
git commit -m "message"
git push
```
  - `add .`代表将目前目录下所有修改过、新创建、删除的文件全放入git暂存区，
  - `commit -m "message"`是将已经用`git add`选中的修改正式保存到本地仓库的历史记录中
