    KaTeX

    KaTeX#

    KaTeX shortcode let you render math typesetting in markdown document. See KaTeX

    Override KaTeX initialization config
    To override the initialization config for KaTeX, create a katex.json file in your assets folder!

    Example#

    Activation#

    KaTeX is activated on the page by first use of the shortcode or render block. you can force activation with empty {{< katex />}} and use delimiters defined in configuration in assets/katex.json.

    Rendering as block#

    1{{< katex display=true >}}
2f(x) = \int_{-\infty}^\infty\hat f(\xi)\,e^{2 \pi i \xi x}\,d\xi
3{{< /katex >}}
    1```katex
2f(x) = \int_{-\infty}^\infty\hat f(\xi)\,e^{2 \pi i \xi x}\,d\xi
3```
    1$$

    $$ f(x) = \int_{-\infty}^\infty\hat f(\xi),e^{2 \pi i \xi x},d\xi $$

    Rendering inline#

    When KaTeX is active on the page it is possible to write inline expressions.

    Code Output
    {{< katex >}}\pi(x){{< /katex >}} \( \pi(x) \)
    \\( \pi(x) \\) \( \pi(x) \)

    Configuration#

    KaTeX configuration could be adjusted by editing assets/katex.json file. For example to enabled inline delimiters $..$ put content below into the file.

    1{
2  "delimiters": [
3    {"left": "$$", "right": "$$", "display": true},
4    {"left": "$", "right": "$", "display": false},
5    {"left": "\\(", "right": "\\)", "display": false},
6    {"left": "\\[", "right": "\\]", "display": true}
7  ]
8}
