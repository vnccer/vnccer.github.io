                              一、整体流程概述#

                              1.1 准备仓库#

                              github中创建名为用户名.github.io的公开仓库，且仓库为空（不要初始化README）

                              1.2 配置Hugo项目#

                              在本地Hugo项目根目录中，打开hugo.toml，将baseURL指向https://用户名.github.io/ 确认theme指向hugo-book

                              1.3 自动化部署（Github Actions）#

                              根目录创建.github/workflows/，且此目录中新建文件deploy.yml，填入Hugo官方推荐的Action配置（Checkout -> Setup Hugo -> Build -> Deploy to GitHub Pages）

                              1.4 开启Github pages#

                              将本地代码推送到github仓库 进入github仓库的settings -> pages build and deployment -> source中选择Githubactions

                              1.5 推送#

                              当向仓库push代码时，github actions自动编译并更新网页 过个2分钟，就可以访问

                              3.2 半自动化部署#

                              vscode中创建rocky666/.github/workflows/，且在workflows/中新建文件hugo.yml，内容复制官方文档https://github.com/vnccer/vnccer.github.io/new/main?filename=.github%2Fworkflows%2Fhugo.yml&workflow_template=pages%2Fhugo

                              接着在github仓库setting中，修改Actions，General，Workflow permissions 修改为读写权限 在Pages中，将建造和部署下，将源修改为Action

                              接着在终端执行任务，将脚本发送给Github

                              1git init
2git remote add origin https://github.com/用户名/用户名.github.io.git
3git add .
4git commit -m "添加自动化部署脚本"
5git push -u origin main
                              • git remote add origin xxxURL是用git建立自己网站的远程连接，远程仓库的代号为origin
                              • git push -u origin main告诉git，以后本地的main分支，默认对应远程origin仓库里的main分支，这个命令成功运行过一次，git就会记住
                              • 查看远程仓库名：git remote
                              • 查看关联的github详细地址git remote -v，其中fetch指从哪里拉取代码，push指推送到哪里
                              • 查看分支追踪关系git remote show origin，

                              以后无论本地更新什么都是三板斧

                              1cd D:\4work\2note\website666\rocky666\
2git add .
3git commit -m "message"
4git push
                              • add .代表将目前目录下所有修改过、新创建、删除的文件全放入git暂存区，
                              • commit -m "message"是将已经用git add选中的修改正式保存到本地仓库的历史记录中

                              5.1 Markdown文档#

                              5.1.1 文档标准配置(Front Matter)(元数据区域)#

                              _index.md文档：

                              1---
2title: "目录名称"               # 目录显示的名称
3bookCollapseSection: true    # 开启该章节的折叠功能
4weight: 1                   # 排序权重，数字越小越靠前
5---

                              具体的markdown文件开头：

                              1---
2title: "主题"  # 侧边栏显示的文字
3date: 2026-03-11             # 文档日期
4draft: false                 # 必须为 false 才能在正式环境中显示
5weight: 1
6---
