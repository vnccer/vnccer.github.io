    一、资产范围#

    特别注意，以下事项请认真查看并留意。
1）漏洞报告需认真填写，标题如“某系统某漏洞”等需写明具体系统与漏洞。
举例：
a）联想商城搜索处存在反射型xss（正确）
b）某系统反射型xss（错误）

2）报告中需提供涉及到风险点的链接（文字版可复制）
举例：
a）漏洞报告提供了可复制的漏洞链接，与相应的证明截图，外加条理清晰的描述文字。（正确）
b）漏洞报告中只提供了截图和过少的文字描述，并未提供实际的风险点链接。（错误）

PS：联想全资子公司、联想投资、联想控股的公司的漏洞，以及与联想完全无关的漏洞，不予审核通过。
如：阳光雨露资产下漏洞暂不收录：*.sunits.com

安想、智慧医疗相关资产暂不收录，域名接管问题内部整改中暂不收录

    二、爬虫#

    2.1 crawlergo + Burp suite#

    点击学习crawlergo

    添加参数，让流量走burp suite

    .\crawlergo_win_amd64.exe -c D:\3patience\chromium\chrome.exe -t 10 --request-proxy http://127.0.0.1:8080 https://baiying.com.cn/

    结果流量一走burp suite，只能扫描出两个网站，后面才发现，是上游代理没删除，流量被转发到7897和7890端口，直接断流，修复后，将 powershell 内容保存到 2026-06-15.txt 文档

    获得44个结果、710个请求、11个子域名、20个关联域名

    可以分类如下：

    攻击面大类 涉及的具体路径/URL 核心测试逻辑与漏洞挖掘思路
    认证、鉴权与高权限接口
    • /auth
    • /supplier
    • /partner
    • /join-us
    		 1. 越权与未授权访问： 测试直接访问 /supplier（供应商）或 /partner（合作伙伴）后台路由，看是否存在前端路由鉴权绕过。
    2. 登录/注册安全： /auth 页面通常包含登录或 Token 验证，测试是否存在任意账号注册、短信/邮箱验证码绕过、密码重定向漏洞或撞库风险。
    敏感数据提交与输入点
    • /recruit/apply
    • /mobile/recruit/apply
    • /demo
    • /mobile/demo
    		 1. 任意文件上传： /recruit/apply（应聘申请）通常带有简历上传功能，重点测试是否能绕过黑白名单限制上传 Webshell、HTML（导致存储型XSS）或恶意 PDF/Docx 文件。
    2. 盲打 XSS / 逻辑漏洞： 申请表单的输入框通常会提交到后台管理系统，可在姓名、经历等字段嵌入 XSS Payload 盲打后台；同时注意测试是否能越权查看他人的应聘简历或演示申请。
    前端框架与敏感密钥泄露
    • /_nuxt/
    • /application/json
    • /sentryTest
    		 1. 前端源码还原： 网站使用 Nuxt.js 架构，通过 /_nuxt/ 路径寻找打包的 JS 文件及关联的 Source Map，还原前端源码，搜寻隐藏的 API 接口或硬编码的第三方 AccessKey/Secret。
    2. 调试点泄露： /sentryTest 属于典型的开发/测试残留接口，需确认是否暴露了 Sentry 系统的 DSN 密钥、内部报错日志或敏感组件版本信息。
    文件处理与目录穿越
    • /driveDownload
    • /download
    • /fusion-download
    		 1. 任意文件下载/读取： 关注这些下载接口在实际传参时（如 ?file=、?path= 或 ?id=），是否对路径做了严格限制。尝试使用 ../ 进行目录穿越，读取服务器敏感配置文件（如 /etc/passwd、web.xml）。
    2. 越权下载： 测试是否能通过修改 ID 序号下载其他企业或用户的专属固件/文件。
    业务逻辑与常规注入点
    • /news/detail
    • /news/list
    • /warranty-enquiry
    • /equipment-procurement/sparePart
    • /repair-service/datasRecovery
    		 1. SQL 注入 / 逻辑绕过： /news/detail（新闻详情）和 /warranty-enquiry（保修查询）必定涉及数据库交互。针对其传递的参数（如新闻 ID、保修单号）进行 SQL 注入、宽字节注入测试。
    2. 逻辑漏洞： 保修查询处是否存在查询遍历，导致批量泄露联想用户的设备序列号及隐私。
    移动端特有资产
    • /mobile
    • /mobile/recruit
    • /mobile/agent-app-download
    		 1. 移动端独立接口： H5 页面（/mobile）由于适配需要，往往会调用一套与 PC 端不同的独立 API 接口，这套接口防御往往相对薄弱，更易出现越权。
    2. APP 逆向： /agent-app-download 提供了安装包下载，可将 APK 下载后进行反编译，提取其中的硬编码域名、内部 API 以及加密算法。
    其他资源与基础响应
    • /image/png
    • /service-policy
    • /feeStandard
    • /service-station
    		 1. SSRF / 信息泄露： 检查静态策略页面或服务站查询中是否包含动态交互参数。
    2. 导航超时分析： 日志中 http://baiying.com.cn/ 出现 Maps timeout，需关注该端口是否存在端口暴露、畸形报文导致拒绝服务（DoS），或强制跳转逻辑缺陷。
