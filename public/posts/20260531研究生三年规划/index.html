    研究生规划

    1005实验室三年发展思路#

    研一#

    研一上学期：搞网安基础#

    • 核心任务： 搞定课程修读 + 补网安基础。
    • 具体规划：
      • 基础通关： 课余时间以 DVWA / Mutillidae 等靶场为切入点，迅速吃透 OWASP Top 10（经典Web漏洞原理及防御）。
      • 技术选栈： 在学期中旬前，结合自身兴趣选定一个网安细分技术栈：
        • Web安全 / 渗透测试（重在漏洞挖掘与利用思路）
        • 二进制逆向 / Pwn（重在底层原理、汇编与内存安全）
        • 红蓝对抗 / 攻防演练（重在内网渗透、域控与免杀）
        • 安全研发 / 白盒审计（重在代码安全、SDL与工具开发）
      • 实战进阶： 根据选定的方向，全力去刷 Hack The Box (HTB)、TryHackMe 或组队参加 CTF 比赛，以赛代练。

    研一下学期：搞学术#

    • 核心任务： 文献阅读 + 确定大方向 + 技能长线积累。
    • 具体规划：
      • 学术破局： 鉴于毕业要求变化，本学期必须全力主攻小论文。大量精读顶会/顶刊文献，选定具体的网安研究方向。
      • 长线规划（三选一）： 根据自己的未来意向：
        • 找工作： 启动 LeetCode 刷题计划（算法是所有大厂研发/安全岗的硬通货），巩固计算机网络和操作系统基础。
        • 考公/体制内：
        • 深造（申博）：

    研二#

    研二上学期：搞论文+日常实习（如果找工作）#

    • 核心任务： 论文产出 + 提前适应招聘节奏。
    • 具体规划：
      • 论文撰写与发表： 动笔撰写小论文。如果实验顺利，争取在 12月前提交或发表至 CCF B/C类会议或期刊
      • 实习试水： 9月一开学就尝试投递日常实习。此阶段目的不在于立刻去上班，而是为了：
        1. 检验自己的简历和技术成色；
        2. 提前适应大厂/网安头部企业的面试节奏与高频考点。 注：如果有合适机会，可去实习体验；若未找到，则全神贯注主攻论文。

    研二下学期：搞暑期实习（如果找工作）#

    • 核心任务： 春招 / 暑期实习
    • 具体规划：
      • 春招与暑期实习： 这是研究生阶段最重要的求职转折点。全力投递大厂的暑期实习生招聘。
      • 意义所在： 暑期实习往往伴随着极高的转正率。拿到转正 Offer 意味着你将在研三开学前直接提前通关。
      • 准备工作： 疯狂复盘之前的 CTF 经历、靶场经验以及 LeetCode 错题，打磨简历。

    研三#

    研三上学期：搞秋招（如果找工作）#

    • 核心任务： 搞 Offer
    • 具体规划：
      • 秋招冲刺： 9月-11月是秋招正式批的黄金期。无论手里是否有转正 Offer，积极参与各大互联网、网安头部企业、国央企、银行国企的校招。
      • 心态管理： 这期间面试密度高、压力大，注意调整心态，实验室成员之间要互通有无、共享内推资源。

    研三下学期：搞大论文#

    • 核心任务： 顺利毕业 + 实验室传承
    • 具体规划：
      • 毕业大论文： 准备毕业答辩，确保顺利毕业。
      • 实验室传承： 在一切顺利的前提下，梳理并捋顺自己负责的实验室研究线路。将自己的代码、工具、文献笔记进行交接，完善实验室的该技术线路，以便研一、研二的师弟师妹们能够完美继承并继续发扬光大。
